Missing Indicator — Missingness ko Feature Banao!
+30 XP on scroll
Core Idea — Ek Line Mein
Missing Indicator = ek naya binary column add karo jo bataye ki original column mein value missing thi ya nahi.

Iska matlab: NaN ko fill bhi karo aur ek alag column mein record karo ki wahan NaN tha — taaki model khud seekh sake ki missing hona kya signal deta hai!
Sirf Fill Karna Kyun Kabhi Kaafi Nahi Hota?
Jab tum mean ya most_frequent se fill karte ho, model ko pata nahi chalta ki woh value original thi ya fill ki gayi.

Lekin missingness khud ek pattern ho sakti hai — jaise:
→ High-income log income field blank chodh dete hain (MNAR)
→ Sick patients test results nahi dete
→ Fraud transactions mein location missing hoti hai

Agar ye pattern hai, toh is signal ko hata dena = information kho dena!
Kaise Kaam Karta Hai?
①
Original Data
Age: [25, NaN, 30, NaN, 22]
Age column mein 2 missing values hain. Seedha fill karne pe information kho jaati hai.
Problem: Why was it missing?
②
Indicator Column Add
Age_Missing: [0, 1, 0, 1, 0]
Naya binary column banao — 1 matlab "yahan NaN tha", 0 matlab "value thi".
Signal preserved!
③
Fill + Keep Both
Age: [25, 25.6, 30, 25.6, 22]
Age_Missing: [0, 1, 0, 1, 0]
Original column bhi fill karo, indicator bhi rakho. Model dono use kar sakta hai!
Best of both worlds!
Real India Example — HDFC Bank Loan Applications
HDFC ke loan application mein Monthly Income field often missing hota hai.
→ Agar income missing hai → ho sakta hai freelancer / self-employed hain (higher risk!)
→ Agar income fill hai → salaried employee hai (lower risk)

SimpleImputer sirf NaN fill karega. Missing Indicator + Imputation dono karo toh model seekhega ki "income missing hona" khud ek risk signal hai!
Missing Indicator Kab Use Karein?
✅ MNAR (Missing Not At Random) data ho
✅ Missing pattern meaningful ho (fraud, risk)
✅ Missing % 5–40% range mein ho
✅ Tree-based models use kar rahe ho (RF, XGBoost)
✅ Business domain knowledge bata rahi ho ki missing = signal
Kab Use Mat Karein?
❌ MCAR data — missingness completely random ho
❌ Very low missing % (< 1%) — extra column pointless
❌ Bahut saari columns hain → feature explosion
❌ Linear models — collinearity issues ho sakti hain
❌ Missing % > 70% — indicator bhi zyada noise dega

import pandas as pd
import numpy as np

# ─── Inline dataset: Student exam data ───
data = {
    'Name':    ['Aarav', 'Priya', 'Ravi', 'Sneha', 'Karan', 'Divya', 'Mohit', 'Ananya'],
    'Age':     [22,      np.nan,  25,      23,      np.nan,  21,      24,      np.nan  ],
    'Score':   [85,      90,      np.nan,  78,      88,      np.nan,  72,      95      ],
    'Income':  [30000,   np.nan,  45000,   np.nan,  28000,   np.nan,  52000,   np.nan  ],
    'City':    ['Mumbai','Delhi', np.nan,  'Surat', 'Mumbai',np.nan,  'Delhi', 'Pune'  ],
}
df = pd.DataFrame(data)

print("Original Data:")
print(df)
print("\nMissing values:\n", df.isnull().sum())

# ─── Step 1: Indicator columns banao ───
# har column ke liye jahan NaN hai, 1 lagao, nahi toh 0
df['Age_Missing']    = df['Age'].isnull().astype(int)
df['Score_Missing']  = df['Score'].isnull().astype(int)
df['Income_Missing'] = df['Income'].isnull().astype(int)
df['City_Missing']   = df['City'].isnull().astype(int)

# ─── Step 2: Original columns fill karo ───
df['Age'].fillna(df['Age'].mean(), inplace=True)
df['Score'].fillna(df['Score'].median(), inplace=True)
df['Income'].fillna(df['Income'].median(), inplace=True)
df['City'].fillna(df['City'].mode()[0], inplace=True)

print("\nAfter Missing Indicator + Imputation:")
print(df[['Name', 'Age', 'Age_Missing', 'Income', 'Income_Missing']])
# Age_Missing = 1 wali rows mein imputed value hai
# Model seekhega: "Age_Missing=1 waale students ki pattern kaisi hai?"

import pandas as pd
import numpy as np
from sklearn.impute import SimpleImputer, MissingIndicator

# ─── Same inline dataset ───
data = {
    'Age':    [22,  np.nan, 25,  23,  np.nan, 21,  24,  np.nan],
    'Score':  [85,  90,     np.nan, 78, 88,  np.nan, 72, 95   ],
    'Income': [30000, np.nan, 45000, np.nan, 28000, np.nan, 52000, np.nan],
}
df = pd.DataFrame(data)

# ─── MissingIndicator — sklearn ka official tool ───
indicator = MissingIndicator(features='missing-only')
# features='missing-only' → sirf unhi columns ke liye indicator banao jinmein NaN ho

indicator_array = indicator.fit_transform(df)
print("Indicator array shape:", indicator_array.shape)  # (8, 3) — 3 columns mein NaN tha

# Indicator DataFrame banao
indicator_cols = [df.columns[i] + '_Missing' for i in indicator.features_]
df_indicator = pd.DataFrame(indicator_array.astype(int), columns=indicator_cols)

# ─── Original columns impute karo ───
imputer = SimpleImputer(strategy='mean')
df_imputed = pd.DataFrame(
    imputer.fit_transform(df),
    columns=df.columns
)

# ─── Dono merge karo ───
df_final = pd.concat([df_imputed, df_indicator], axis=1)

print("\nFinal DataFrame with Indicators:")
print(df_final.round(1))
print("\nLearned missing column indices:", indicator.features_)
# [0, 1, 2] → sab teeno columns mein NaN tha

Random Sample Imputation — Distribution Preserve Karo!
+30 XP on scroll
Core Idea — Ek Line Mein
Random Sample Imputation = missing value ko fill karo usi column ke ek randomly selected existing (non-missing) value se — mean ya mode nahi!

Ye method original data ki distribution preserve karta hai — mean/median imputation jo distribution ko artificially peak kar deta hai, woh problem yahan nahi hoti.
Mean Imputation vs Random Sample Imputation
Mean Imputation — Kya Hota Hai
Original data: [10, 90, 20, NaN, 80, 15, NaN, 85]

Mean = (10+90+20+80+15+85) / 6 = 50

After fill: [10, 90, 20, 50, 80, 15, 50, 85]

❌ Real distribution bimodal thi (low + high values). Mean = 50 jo kabhi aaya hi nahi! Distribution distort ho gayi.
Random Sample — Distribution Bachti Hai
Original data: [10, 90, 20, NaN, 80, 15, NaN, 85]

Random pick karo existing values se: 20, 80

After fill: [10, 90, 20, 20, 80, 15, 80, 85]

✅ Values real data se aayi hain! Distribution ka shape preserved hai — bimodal pattern intact!
Real India Example — IPL Batting Data
Imagine karo ek IPL dataset mein batsman ke Strike Rate column mein kuch values missing hain.

→ Mean imputation: Sab NaN → average strike rate (say 135) se fill. Ye aggressive hitters aur defensive hitters dono ke liye same value deta hai — wrong!
→ Random Sample: Koi NaN → kisi existing batsman ka actual strike rate (e.g., 162 ya 108) se fill. Distribution real rehti hai — kuch high, kuch low — exactly jaise real cricket data hota hai!
Ek Important Note
Random Sample Imputation non-deterministic hai — har baar run karne par alag result aayega!
Solution: np.random.seed(42) set karo taaki reproducible results milein. Production mein hamesha seed fix karo!

Random Sample Imputation — Complete Python Code
🐍 Python
import pandas as pd
import numpy as np

# ─── Inline dataset: IPL Batting Stats ───
np.random.seed(42)
data = {
    'Player':       ['Rohit', 'Virat', 'Dhoni', 'Pant', 'Jadeja',
                     'Bumrah', 'Shami', 'KL Rahul', 'Hardik', 'Surya'],
    'Strike_Rate':  [130, np.nan, 135, 142, np.nan,
                     np.nan, 120, 145, np.nan, 170   ],
    'Avg_Score':    [38,  49,    np.nan, 32, 22,
                     12,  np.nan, 45,  35,  np.nan  ],
    'Matches':      [200, 215, 250, 100, 180,
                     120, np.nan, 105, 90, np.nan   ],
}
df = pd.DataFrame(data)

print("Original Data:")
print(df)
print("\nMissing values:\n", df.isnull().sum())

# ─── Random Sample Imputation Function ───
def random_sample_impute(df, col, random_state=42):
    """Ek column ke NaN ko usi column ki random non-null values se fill karo."""
    np.random.seed(random_state)

    null_count = df[col].isnull().sum()

    if null_count == 0:
        return df  # koi NaN nahi — kuch karne ki zaroorat nahi

    # Step 1: existing (non-null) values se random sample lo
    random_values = df[col].dropna().sample(
        n=null_count,         # utne hi values jitne NaN hain
        replace=True,          # replacement ke saath (zyada NaN hone par bhi kaam kare)
        random_state=random_state
    ).values

    # Step 2: NaN wali positions fill karo
    df_copy = df.copy()
    df_copy.loc[df_copy[col].isnull(), col] = random_values

    return df_copy

# ─── Sab columns pe apply karo ───
num_cols = ['Strike_Rate', 'Avg_Score', 'Matches']
df_imputed = df.copy()

for col in num_cols:
    df_imputed = random_sample_impute(df_imputed, col)

print("\nAfter Random Sample Imputation:")
print(df_imputed)

# ─── Distribution compare karo ───
print("\nStrike_Rate — Original Non-Null Stats:")
print(df['Strike_Rate'].dropna().describe().round(1))

print("\nStrike_Rate — After Imputation Stats:")
print(df_imputed['Strike_Rate'].describe().round(1))
# Mean aur std roughly same rahenge — distribution preserved!
🎲 Random Sample Imputation ka magic: mean/std imputation ke baad change ho jaata hai, random sample ke baad roughly same rehta hai! Distribution ka shape preserve hota hai.

KNN Imputer — Apne Neighbors se Seekho!
+30 XP on scroll
Core Idea — Ek Line Mein
KNN Imputer = missing value ko fill karo similar rows (neighbours) ki values ka average leke — ye multivariate imputation hai!

Pehle ke methods ek column ko independently dekh ke fill karte the. KNN sabhi columns ki information ek saath use karta hai taaki best fill value dhundhe.
KNN Kaise Kaam Karta Hai? — Analogy
Sochte hain tum ek naya neighbourhood mein shift hue. Tumhe nahi pata wahan ghar ka rent kya hoga.

SimpleImputer (mean): Poore city ka average rent lo → rough estimate
KNN Imputer: Tumhare similar neighbours dhundho (same size ghar, same area, same floor) → unka average lo → zyada accurate!

Yahi KNN karta hai — similar rows = neighbours, unki non-missing values se missing value fill karta hai!
Univariate vs Multivariate Imputation
🔢
Univariate (SimpleImputer)
Sirf ek column dekh ke fill karo.

Age NaN hai? → Age column ka mean lo → fill karo.

Salary information ignore! Height information ignore!

Like asking: "Sheher mein average age kya hai?"
🤝
Multivariate (KNN Imputer)
Saari columns ek saath dekho.

Age NaN hai? → Salary, Height, City sab dekho → similar rows dhundho → unki Age ka average lo!

Like asking: "Similar salary, height, city wale logon ki age kya hai?"
Real India Example — Swiggy Delivery Riders
Swiggy ke delivery riders ka data hai: Rating, Deliveries_Per_Day, Experience_Years, Earnings.

Ek rider ki Rating missing hai.

→ SimpleImputer (mean): Sab riders ka average rating (say 4.1) dedo — inaccurate!
→ KNN Imputer: Isko same level ke riders dhundo — similar deliveries per day, similar experience → unka rating 4.6 hai → fill karo 4.6 se!

Result: Experienced high-volume rider ko 4.6 milega, not the global average of 4.1. Zyada realistic!

KNN Imputer — Step-by-Step & Python Code
KNN Imputation — Andar Se Kya Hota Hai?
📊 Data Standardize Karo
KNN distances calculate karta hai — isliye scale matter karta hai!
Age (22–65) aur Salary (20,000–5,00,000) ek saath distance mein — Salary dominate karega.
KNNImputer internally standardization handle karta hai. Phir bhi pehle StandardScaler lagana good practice hai.
📏 Distance Calculate Karo
NaN wali row ke liye — baaki sabhi rows ke saath Euclidean distance nikalo (sirf available/non-NaN columns ke basis par).
Row A: [Age=25, Salary=40k, NaN] → Row B: [Age=27, Salary=42k, Rating=4.2] → distance = √((25-27)² + (40k-42k)²)
🏆 K Nearest Neighbours Dhundho
Sabse kam distance wali K rows select karo — ye hain neighbours!
Default: n_neighbors=5 matlab 5 nearest rows.
K=5 means: 5 most similar riders ke basis par missing rating fill karenge.
🧮 Weighted Average Lo
K neighbours ki missing column ki values ka weighted average nikalo.
Jo neighbour closer hai (kam distance), uska weight zyada hoga — uniform ya distance-based weighting.
Result: Distance-based fill → similar neighbours ka zyada influence!
✅ Fill Karo
Calculated average se NaN replace karo. Har NaN ke liye ye process repeat hoti hai.
Important: fit() train data pe, transform() dono train aur test pe — data leakage nahi hoga!
KNN Imputer — Complete Python Code
🐍 Python — KNN Imputer
import pandas as pd
import numpy as np
from sklearn.impute import KNNImputer

# ─── Inline dataset: Swiggy Delivery Riders ───
data = {
    'Rider':           ['Raju', 'Suresh', 'Amit', 'Vikram', 'Pooja',
                        'Kiran', 'Deepak', 'Meena', 'Arjun', 'Sunita'],
    'Deliveries_Day':  [18,   25,      np.nan, 30,      15,
                        22,   28,      np.nan, 19,      26     ],
    'Exp_Years':       [3,    5,       4,      6,       np.nan,
                        4,    5,       3,      np.nan,  5      ],
    'Avg_Distance_km': [8.5,  12.0,    10.5,   np.nan,  7.0,
                        9.5,  np.nan,  8.0,    9.0,     11.5   ],
    'Rating':          [4.2,  4.7,     np.nan, 4.9,     4.0,
                        4.4,  4.8,     4.1,    np.nan,  4.6    ],
}
df = pd.DataFrame(data)
df_nums = df.drop('Rider', axis=1)   # sirf numerical columns

print("Original Data (numerical only):")
print(df_nums)
print("\nMissing values:\n", df_nums.isnull().sum())

# ─── KNN Imputer with k=3 ───
knn_imputer = KNNImputer(
    n_neighbors=3,           # 3 nearest neighbours use karo
    weights='distance'       # closer neighbours ka zyada weight
)

df_imputed = pd.DataFrame(
    knn_imputer.fit_transform(df_nums),
    columns=df_nums.columns
)

print("\nAfter KNN Imputation (k=3):")
print(df_imputed.round(2))

# ─── Before vs After — specific row check ───
print("\nAmit (row 2) — Rating was NaN:")
print(f"  Original: {df.loc[2, 'Rating']}")
print(f"  KNN fill: {df_imputed.loc[2, 'Rating']:.2f}")
# Amit ke 3 nearest neighbors (similar deliveries, exp, distance) ki rating use ki!
🤝 KNN Imputer ne Amit ki Rating Suresh (4.7), Kiran (4.4), Deepak (4.8) se fill ki — ye sab similar Deliveries_Day aur Exp_Years wale riders hain! Global mean (4.4) se zyada meaningful!
SimpleImputer vs KNN Imputer — Side-by-Side
🐍 Python — Comparison
import pandas as pd
import numpy as np
from sklearn.impute import SimpleImputer, KNNImputer

# ─── Same riders dataset ───
data = {
    'Deliveries_Day':  [18, 25, np.nan, 30, 15, 22, 28, np.nan, 19, 26],
    'Exp_Years':       [3,  5,  4,      6,  np.nan, 4, 5,  3,      np.nan, 5],
    'Rating':          [4.2, 4.7, np.nan, 4.9, 4.0, 4.4, 4.8, 4.1, np.nan, 4.6],
}
df = pd.DataFrame(data)

# ─── Method 1: SimpleImputer (mean) ───
mean_imputer = SimpleImputer(strategy='mean')
df_mean = pd.DataFrame(
    mean_imputer.fit_transform(df),
    columns=df.columns
)

# ─── Method 2: KNNImputer ───
knn_imputer = KNNImputer(n_neighbors=3)
df_knn = pd.DataFrame(
    knn_imputer.fit_transform(df),
    columns=df.columns
)

# ─── Compare results ───
print("Row 2 (Amit — Deliveries missing, Rating missing):")
print(f"  Original:        Deliveries=NaN,  Rating=NaN")
print(f"  Mean Imputed:    Deliveries={df_mean.loc[2,'Deliveries_Day']:.1f}, Rating={df_mean.loc[2,'Rating']:.2f}")
print(f"  KNN Imputed:     Deliveries={df_knn.loc[2,'Deliveries_Day']:.1f},  Rating={df_knn.loc[2,'Rating']:.2f}")

print("\nRow 4 (Pooja — Exp_Years missing):")
print(f"  Mean Imputed:    Exp_Years={df_mean.loc[4,'Exp_Years']:.2f}")
print(f"  KNN Imputed:     Exp_Years={df_knn.loc[4,'Exp_Years']:.2f}")
# KNN: Pooja ki Deliveries (15) aur Rating (4.0) jaisi rows dhundi
# → unka Exp_Years (2-3 years likely) use kiya
# Mean: sab ka average (avg~4.3 years) — less accurate for Pooja!
📊 KNN vs Mean: Junior riders (low deliveries, low rating) ke liye KNN kam experience deta hai — mean imputation sab ko global average de deta hai. KNN context-aware hai!
🤔 KNN Imputer SimpleImputer se fundamentally alag kyu hai?
AYe NaN wali rows ko dataset se delete karta hai
BYe categorical aur numerical dono columns ke liye equally kaam karta hai
CYe missing value fill karne ke liye sabhi columns ki information use karta hai — sirf ek column nahi
DYe SimpleImputer se computationally faster hai

KNN — Sahi K Choose Karo & Pitfalls
+30 XP on scroll
K ka Value Kaisa Choose Karein?
n_neighbors (K) — ye ek important hyperparameter hai!
K=1
Too Small
Sirf 1 neighbour
Noisy! Nearest row ka value directly use karo — agar woh row outlier ho toh fill bhi galat hogi.
❌ Overfitting to single point
K=5
Default (Good!)
5 neighbours (default)
Balance of local pattern aur noise reduction. Chhote datasets ke liye generally best starting point.
✅ sklearn default — shuruaat yahan se karo
K=20+
Too Large
Bahut saare neighbours
Blur ho jaata hai — local patterns miss hoti hain. Bade K pe result mean imputation jitna hi ho jaata hai!
⚠️ Underfitting to global mean
Practical Rule of Thumb
→ Small dataset (<500 rows): K = 3–5
→ Medium dataset (500–5000 rows): K = 5–10
→ Large dataset (>5000 rows): K = 10–20

General formula: K ≈ √n jahan n = dataset mein rows ki sankhya. Phir try karo 2-3 values aur results compare karo!
KNN ke Common Pitfalls
Scale ka Issue — Sabse Bada Trap!
Agar columns ka scale alag hai (Age: 0–100, Salary: 0–10,00,000), toh Salary distance mein dominate karega.
Age ka effect practically zero ho jaayega → galat neighbours milenge.
Fix: StandardScaler ya MinMaxScaler pehle lagao, phir KNN Imputer use karo!
Speed — Large Datasets pe Slow
KNN har NaN ke liye sabhi rows ke saath distance calculate karta hai → O(n²) complexity.
10,000 rows × 50 columns → bahut slow ho sakta hai!
Fix: Chhota subset le ke test karo pehle. Ya SimpleImputer use karo agar speed priority ho.
Categorical Columns — Direct Support Nahi
KNNImputer sirf numerical data ke saath kaam karta hai by default.
Categorical columns (City, Gender) ke liye encoding zaroori hai.
Fix: Categorical columns ko pehle OrdinalEncoder se encode karo, tab KNN lagao. Ya unke liye SimpleImputer separately use karo.
Bahut Zyada NaN — Accuracy Giregi
Agar ek row mein 80% values missing hain, toh distance calculation unreliable ho jaati hai — similar rows galat milenge.
Fix: Agar kisi row mein > 50% values missing hon, woh row drop karna zyada practical hai.
🤔 KNN Imputer mein n_neighbors=1 set karne ka kya problem hai?
AEk akela nearest neighbour outlier ho sakta hai — imputation noise-sensitive ho jaata hai
BK=1 pe computation bahut slow ho jaata hai
Csklearn error throw karta hai invalid K ke liye
DK=1 mean imputation jaisi results deta hai

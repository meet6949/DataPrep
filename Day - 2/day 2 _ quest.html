Working with CSV Files
+30 XP on scroll
CSV kya hota hai?
CSV (Comma-Separated Values) = ek simple text file jisme data rows aur columns mein hota hai — commas se alag kiya hua.

Real-world analogy: Excel spreadsheet ko plain text mein save karo — wahi CSV hai!
90% real-world datasets CSV format mein milte hain — Kaggle, government data, company exports — sab CSV!
Golden Rule
CSV pehle pandas mein load karo, phir explore karo — kabhi bhi raw file manually mat edit karo! Ek galat comma poora dataset corrupt kar sakta hai.
CSV kab use karein?
✅ Simple tabular data (rows + columns)
✅ Data share karna hai kisi bhi tool se
✅ Small to medium datasets (< 1 GB)
✅ Machine learning training data
CSV kab use na karein?
❌ Nested/hierarchical data (JSON better)
❌ Very large datasets (> 1 GB) — Parquet use karo
❌ Multiple related tables — SQL better
❌ Binary data (images, audio)
Python mein CSV — Basics se Advanced

JSON vs SQL — Kab kya use karein?
JSON (JavaScript Object Notation) = nested/hierarchical data ke liye — APIs, configs, NoSQL databases.
SQL (Structured Query Language) = relational tables ke liye — customer + orders + products linked data.

Real analogy: JSON = Telegram message (free format) | SQL = Bank ledger (strict rows & relations)
JSON Format — Kaisa dikhta hai?

Python mein JSON — Read, Parse & Flatten
JSON File Read & Parse karo
🐍 Python 
import json
import pandas as pd

# ─── JSON file read karo ───
with open('swiggy_orders.json', 'r', encoding='utf-8') as f:
    data = json.load(f)      # Python dict/list ban jaata hai

# ─── JSON string parse karo ───
json_string = '{"name": "Rahul", "city": "Surat"}'
parsed = json.loads(json_string)   # loads = "load string"
print(parsed['name'])    # Rahul

# ─── Nested JSON ko flat DataFrame mein convert karo ───
# Agar data list of dicts hai:
df = pd.DataFrame(data)

# Deeply nested ke liye:
from pandas import json_normalize
df = json_normalize(
    data,
    record_path=['items'],          # Nested list column
    meta=['order_id', 'customer']   # Parent fields jo chahiye
)

# ─── DataFrame wapas JSON mein save karo ───
df.to_json('output.json', orient='records', indent=2)
🔑 json.load() = file se | json.loads() = string se — ye confusion aata hai beginners ko! 's' = string!
🤔 json.load() aur json.loads() mein kya fark hai?
Aload() dict deta hai, loads() list deta hai
Bload() bade JSON ke liye, loads() chhote ke liye
Cload() file object se, loads() string se padhta hai
Dload() standard library, loads() third-party library hai

API with Authentication — API Key
🐍 Python 
import requests
import os

# ─── API Key kabhi code mein hardcode mat karo! ───
# Environment variable mein store karo
API_KEY = os.environ.get("OPENWEATHER_API_KEY")

url = "https://api.openweathermap.org/data/2.5/weather"
params = {
    "q": "Mumbai,IN",
    "appid": API_KEY,       # Key parameter mein
    "units": "metric"
}

# Ya Header mein bheji jaati hai (Bearer token)
headers = {"Authorization": f"Bearer {API_KEY}"}
response = requests.get(url, params=params, headers=headers)

if response.status_code == 200:
    data = response.json()
    print(f"City: {data['name']}")
    print(f"Temp: {data['main']['temp']}°C")
    print(f"Weather: {data['weather'][0]['description']}")
elif response.status_code == 401:
    print("API Key galat hai!")
elif response.status_code == 429:
    print("Rate limit exceed! Thoda wait karo.")

Pagination, Rate Limiting & Error Handling
APIs — Real Problems

Beginner API call = crash on error | Pro API call = graceful handling every time!
Pagination — Sab Pages fetch karo
🐍 Python 
import requests
import time

def fetch_all_pages(base_url, api_key, max_pages=10):
    all_records = []
    page = 1

    while page <= max_pages:
        response = requests.get(base_url, params={
            "api_key": api_key,
            "page": page,
            "per_page": 100
        })

        if response.status_code == 200:
            data = response.json()
            records = data.get('results', [])

            if not records:        # Koi data nahi bcha?
                break              # Sab pages done!

            all_records.extend(records)
            print(f"Page {page}: {len(records)} records fetched")
            page += 1
            time.sleep(0.5)        # Rate limiting se bachne ke liye!

        elif response.status_code == 429:
            print("Rate limited! 60 seconds wait...")
            time.sleep(60)         # Retry after wait

        else:
            print(f"Error {response.status_code} — stopping")
            break

    return all_records

# Usage
records = fetch_all_pages("https://api.example.com/products", api_key="ABC123")
print(f"Total fetched: {len(records)}")
🔄 time.sleep() bahut zaroori hai — bina iske production API banned kar deti hai tumhara IP!

Missing Values Handle karo
🐍 Python 
import pandas as pd
import numpy as np

df = pd.read_csv('customer_data.csv')

# ─── Strategy 1: Drop karo (jab bahut kam data missing ho) ───
df.dropna(subset=['customer_id'], inplace=True)   # Key column null nahi hona chahiye

# ─── Strategy 2: Fill with mean/median (numerical) ───
df['age'].fillna(df['age'].median(), inplace=True)       # Age ke liye median better hai
df['salary'].fillna(df['salary'].mean(), inplace=True)

# ─── Strategy 3: Fill with mode (categorical) ───
df['city'].fillna(df['city'].mode()[0], inplace=True)    # Most common city

# ─── Strategy 4: Forward Fill (time series) ───
df['stock_price'].fillna(method='ffill', inplace=True)   # Previous value se fill karo

# ─── Strategy 5: Custom value ───
df['rating'].fillna(0, inplace=True)   # Missing rating = 0

print(df.isnull().sum())   # Check: sab zeros hone chahiye!
⚡ Pro Tip: Medical data mein missing value = patient ne jawab nahi diya — zero se fill mat karo! Context samjho!
Outlier Detection & Treatment
🐍 Python 
import pandas as pd

df = pd.read_csv('housing_prices.csv')

# ─── Method 1: IQR (Interquartile Range) ───
Q1 = df['price'].quantile(0.25)
Q3 = df['price'].quantile(0.75)
IQR = Q3 - Q1

lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR

# Outliers hata do
df_clean = df[(df['price'] >= lower) & (df['price'] <= upper)]
print(f"Removed {len(df) - len(df_clean)} outliers")

# ─── Method 2: Cap karo (hataane ki jagah limit karo) ───
df['price'] = df['price'].clip(lower=lower, upper=upper)

# ─── Method 3: Z-Score ───
from scipy import stats
z_scores = stats.zscore(df['price'])
df_clean = df[abs(z_scores) < 3]  # 3 standard deviations ke andar
📌 Mumbai property prices mein ek ₹50 crore villa = outlier jo average distort kar dega — remove ya cap karo!
Data Type Fixes & String Cleaning
🐍 Python 
import pandas as pd

df = pd.read_csv('messy_data.csv')

# ─── String columns clean karo ───
df['city'] = df['city'].str.strip()      # Leading/trailing spaces hataao
df['city'] = df['city'].str.lower()      # Lowercase — 'Mumbai' == 'mumbai'
df['city'] = df['city'].str.replace('  ', ' ')  # Double spaces fix

# ─── Phone numbers normalize karo ───
df['phone'] = df['phone'].str.replace(r'\D', '', regex=True)  # Non-digits hatao
df['phone'] = df['phone'].str[-10:]      # Last 10 digits rakho

# ─── Date column fix karo ───
df['order_date'] = pd.to_datetime(df['order_date'], errors='coerce')
df['year']  = df['order_date'].dt.year
df['month'] = df['order_date'].dt.month

# ─── Numerical columns fix karo ───
df['price'] = df['price'].str.replace('₹', '').str.replace(',', '')
df['price'] = pd.to_numeric(df['price'], errors='coerce')

# ─── Categorical encoding ───
df['gender'] = df['gender'].map({'Male': 1, 'Female': 0, 'M': 1, 'F': 0})

print(df.dtypes)   # Verify: sab theek hain?
💡 Real data mein 'mumbai', 'Mumbai', 'MUMBAI', 'Mumbaii' sab milte hain — str.lower() + strip() zaroor karo!
